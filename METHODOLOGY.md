# 방법론 (Methodology)

이 문서는 무엇을 등재하고, 어떻게 검증하며, 어떤 기준으로 분류하는지를 정의합니다.
정책은 이 문서에, 그 정책을 기계적으로 강제하는 규칙은 `web/src/content/schemas.ts`(zod)와 `scripts/`에 둡니다.

## 1. 등재 기준

### 1.1 조직 등재 기준 (다음 중 하나 이상)
- AI 모델 또는 AI 기반 제품을 직접 개발한다.
- AI가 제품의 핵심 기능 또는 주요 경쟁력이다.
- 자체 학습·파인튜닝·데이터 처리·추론 또는 배포 기술을 보유한다.
- 공개된 연구·제품·고객 사례 또는 기술 문서가 존재한다.
- 국내 법인·국내 연구조직 또는 한국 연구진이 실질적으로 개발을 주도한다.

**제외 또는 별도 분류:**
- 단순히 외부 생성형 AI API를 연결한 서비스
- AI 기능이 부가 기능에 불과한 일반 소프트웨어
- 출시 근거가 없는 발표 단계 프로젝트
- 공식 출처를 확인할 수 없는 기업
- 광고성 랜딩페이지만 존재하는 서비스

### 1.2 모델 등재 기준
- 실제 공개 또는 상용 출시된 모델
- 공식 발표·모델 카드·기술보고서 또는 논문 존재
- 개발 조직 확인 가능 / 모델 이름·출시 시점 확인 가능
- API·가중치·데모 중 하나 이상 접근 가능

발표만 되고 배포되지 않은 모델은 `announced` 상태로 별도 표시한다.

### 1.3 한국 AI 판정 기준 (다음 중 하나 이상)
- 한국 법인 또는 기관이 개발 주체
- 한국 소재 연구조직이 개발을 주도
- 한국인 연구진이 중심이 된 독립 프로젝트
- 한국 시장과 한국어를 핵심 대상으로 개발
- 국내 조직이 모델의 학습·운영에 실질적 책임을 보유

단순히 한국어를 지원하는 해외 모델은 포함하지 않는다.

## 2. AI 핵심도 분류 (`ai_core_level`)

| 값 | 의미 |
|---|---|
| `foundation_model` | 자체 기반 모델을 개발 |
| `model_builder` | 자체 학습 또는 유의미한 파인튜닝 모델 보유 |
| `ai_native` | AI가 제품 전체의 핵심 |
| `ai_infrastructure` | AI 개발·운영 인프라 제공 |
| `ai_enabled` | 기존 제품에 AI 기능을 통합 |
| `research_only` | 연구 중심 조직 또는 프로젝트 |
| `unverified` | 충분한 근거가 확인되지 않음 |

## 3. 정보 검증 체계

### 3.1 출처 우선순위
1. 공식 논문 및 기술보고서
2. 공식 모델 카드
3. 공식 GitHub 및 Hugging Face
4. 공식 제품 문서
5. 정부·공공기관 자료
6. 기업 공시
7. 신뢰 가능한 언론
8. 기업 보도자료
9. 관계자 제출 자료

> 언론 기사나 보도자료만으로 모델의 기술적 특성을 확정하지 않는다.

### 3.2 검증 상태 (`verification_status`)
| 값 | 의미 |
|---|---|
| `verified` | 복수의 신뢰 가능한 출처로 검증 |
| `official-source` | 공식 출처 1개 이상으로 확인 |
| `community-reviewed` | 커뮤니티 검토 완료 |
| `submitted` | 신규 제보 상태 |
| `needs-review` | 정보 재검토 필요 |
| `outdated` | 마지막 검증일이 기준을 초과 |
| `disputed` | 이해관계자 간 이견 존재 |
| `archived` | 서비스 또는 프로젝트 종료 |

### 3.3 신뢰도 표시
개별 필드에는 필요에 따라 `confident` / `likely` / `speculative` / `undisclosed` / `not-applicable` 신뢰도를 부여한다.
**추정값은 사실값과 동일한 형태로 표시하지 않는다.**

### 3.4 검증 주기
| 유형 | 주기 |
|---|---|
| 모델 | 90일 |
| 기업 및 조직 | 180일 |
| 제품 및 서비스 | 90일 |
| 오픈소스 | 60일 |
| 데이터셋 및 벤치마크 | 180일 |
| 종료 또는 폐업 상태 | 수시 |

`last_verified` + 주기를 초과하면 자동으로 `needs-review`(또는 `outdated`) 라벨을 부여한다.
